2 Standalone mode (single device, no install)

🗂️ Just open the file

For a single device with no network sharing needed, you can open the app directly.

Open: public/index.html in Chrome or Edge

⚠️ In standalone mode the PWA install button will not work and some features (manifest, service worker) require the server. Data is saved in your browser's IndexedDB. For full PWA support, use Server Mode below.

3 Server mode (recommended — multi-device + PWA)

✅ Server mode fixes all PWA diagnostic errors. It gives you HTTPS-equivalent localhost, a working manifest, a controlled service worker, and real-time sync across all devices on your Wi-Fi.

Prerequisites — Install Node.js first

📦 Node.js (required, free)
  1. Go to https://nodejs.org
  2. Download the LTS version (e.g. 20.x LTS)
  3. Run the installer — click Next through all steps
  4. Make sure "Add to PATH" is ticked (it is by default)
  5. Finish the installer, then restart your computer once
ℹ️ To verify Node.js is installed: open Command Prompt and type node -v. You should see something like v20.11.0.

Starting the server

🪟 Windows
🍎 macOS
🐧 Linux

Everything is automated — one double-click is all you need.

  1. Open the TERAHEADBMS-v2.0 folder
  2. Double-click server.bat
  3. First run: it installs Express automatically (~30 sec, needs internet once)
  4. Your browser opens at http://localhost:3000 automatically
  5. Keep the black terminal window open while using the app
  6. To stop: click the terminal window and press Ctrl+C
⚠️ Windows may show a SmartScreen warning ("Unknown publisher"). Click More info → Run anyway. This is normal for unsigned .bat files.
💡 To start the server automatically on Windows startup: Right-click server.bat → Create shortcut → Press Win+R, type shell:startup, move the shortcut into that folder.

Connecting from other devices (phones, tablets, other PCs)

When the server starts, it prints your LAN IP addresses. Open one of those on any device on the same Wi-Fi.

# Example server output:
  Local:    http://localhost:3000
  Network:  http://192.168.1.45:3000   ← type this on other devices
  • All devices must be on the same Wi-Fi network
  • Changes on one device appear on all others in real-time
  • The 🌐 Network badge in the top bar confirms server mode is active
  • If other devices can't connect, check Windows Firewall (see FAQ)

4 Installing as a PWA (desktop / mobile app)

ℹ️ PWA install requires the server to be running (localhost or HTTPS). It does NOT work with file:// URLs.
🖥️ Desktop (Windows / Mac)
  1. Start the server via server.bat
  2. Open Chrome or Edge at http://localhost:3000
  3. Click 📲 Install in the top bar
    or use browser menu (⋮) → "Install TERAHEADBMS"
  4. App appears in Start Menu / taskbar / Dock
📱 Android
  1. Connect phone to same Wi-Fi as the PC running the server
  2. Open Chrome on phone
  3. Go to http://<your-PC-IP>:3000
  4. Tap browser menu (⋮) → "Add to Home Screen"

5 Fixing PWA Diagnostics errors

Open the diagnostics via Settings → PWA Diagnostics. Here's what each error means and how to fix it:

  • ❌
    Protocol: file: — Needs HTTPS or localhost
    Fix: You opened index.html directly. Instead, run server.bat and open http://localhost:3000 in Chrome.
    Once you do this, this check turns ✅ automatically.
  • ⚠️
    Service Worker: Registered, not controlling yet
    Fix: This happens on the very first load. The app automatically reloads itself once to let the SW take control. If it persists: press Ctrl+Shift+R (hard reload) once, then open normally.
  • ❌
    Manifest: Not found at /manifest.json
    Fix: Only works when served via the server. Run server.bat → open http://localhost:3000. The manifest is at public/manifest.json and is served automatically.
  • ⚠️
    Install event captured: Not yet
    Fix: This resolves automatically once protocol and manifest are both ✅. After running the server and reloading once, the 📲 Install button appears in the top bar within a few seconds.
  • ✅
    Browser: Chrome ✅
    Already correct. Chrome and Edge both support PWA install fully.
🎯 Summary: All 5 diagnostics pass automatically once you run server.bat and open http://localhost:3000 in Chrome. No other configuration needed.

6 Data, backup & migration

💾 Where data lives
  • 🌐Server mode:
    thbms_data.json in the server folder. Back this file up regularly.
  • 💻Standalone mode:
    Browser IndexedDB (lives inside Chrome profile). Use Settings → Backup to export a portable JSON.
♻️ Migration (local → server)

If you used the app standalone before, your local IndexedDB data is automatically migrated to the server on first login. No data is lost.

💡 If migration doesn't happen automatically: export a backup from Settings while in standalone mode, then restore it from Settings after launching the server.
📦 Regular backups
  • Go to Settings → Storage & Backup → Download Full Backup regularly
  • A reminder banner appears every 90 days if you haven't backed up
  • Connect Google Drive in Settings for automatic cloud backups
  • To restore: Settings → Restore from Backup → select your .json file
  • Restores are merge-safe — they add missing data without deleting anything

7 Troubleshooting FAQ

📲 Install button still not appearing after running server.bat▾
  1. Make sure you opened http://localhost:3000 in Chrome or Edge (not Firefox/Safari)
  2. Press Ctrl+Shift+R once to hard-reload
  3. Wait 5 seconds — the install event fires after the SW registers
  4. Check Settings → PWA Diagnostics — all 5 checks should now be ✅
  5. If the app is already installed, uninstall it first (the button disappears when installed)
  6. Try opening in an Incognito window — some Chrome flags can block install events
🔥 Windows Firewall blocking other devices▾

When the server starts for the first time, Windows asks if Node.js should be allowed on the network. Click Allow Access.

If you missed that prompt:

  1. Open Windows Defender Firewall (search in Start Menu)
  2. Click Allow an app through firewall
  3. Find Node.js → tick both Private and Public
  4. Or run this in an admin Command Prompt:
    netsh advfirewall firewall add rule name="Node THBMS" dir=in action=allow protocol=TCP localport=3000
⚡ Port 3000 already in use▾

Another program is using port 3000. To change the port:

  1. Open server.js in Notepad
  2. Find line: const PORT = process.env.PORT || 3000;
  3. Change 3000 to another number, e.g. 3001
  4. Save and restart server.bat
  5. Access the app at http://localhost:3001
💾 Data disappeared after restarting server▾

Check if thbms_data.json exists in the server folder. If it's empty or missing:

  • Log in — the app will automatically check your local IndexedDB and migrate data to the server
  • If you have a backup JSON, restore it via Settings → Restore from Backup
  • Prevent this: keep a regular backup (Settings → Download Full Backup)
🔄 How to update to a new version of TERAHEADBMS▾
  1. Stop the server (Ctrl+C)
  2. Keep thbms_data.json — do not delete it
  3. Replace public/index.html with the new version's HTML file
  4. Restart server.bat
  5. Press Ctrl+Shift+R in the browser once to clear the old cached app
🖨️ Setting up a thermal printer (QZ Tray)▾
  1. Download and install QZ Tray from qz.io
  2. Run QZ Tray (it appears in the system tray)
  3. In TERAHEADBMS, go to POS → complete a sale → receipt appears
  4. Click the 🖨️ Print button — you'll be prompted to select a printer
  5. Click "Save as default" to avoid the picker next time
  6. You can also set the default printer in Settings → QZ Tray Printer
